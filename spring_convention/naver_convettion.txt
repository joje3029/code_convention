# 네이버 Java 코딩 컨벤션 정리

## 1. 파일 공통 요건
- 파일 인코딩: UTF-8 사용
- 새줄 문자: LF(Line Feed, 0x0A) 사용
- 파일 끝에는 새줄 문자 필수

## 2. 이름 규칙(Naming)
- 식별자: 영문/숫자/언더스코어만 허용
- 한국어 발음대로 표기 금지
- 패키지명: 소문자로만 구성
- 클래스/인터페이스명: 대문자 카멜케이스
- 메서드명: 소문자 카멜케이스, 동사/전치사로 시작
- 상수: 대문자와 언더스코어로 구성
- 변수: 소문자 카멜케이스
- 임시 변수 외 1글자 이름 사용 금지

## 3. 선언(Declarations)
- 소스파일당 탑레벨 클래스 1개만 포함
- static import에만 와일드카드 허용
- 한 줄에 한 문장만 작성
- 하나의 선언문에는 하나의 변수만
- 배열 대괄호는 타입 뒤에 선언

## 4. 들여쓰기(Indentation)
- 하드탭 사용
- 탭 크기: 4개의 스페이스
- 블록 들여쓰기 필수

## 5. 중괄호(Braces)
- K&R 스타일로 중괄호 선언
- else, catch, finally, while는 닫는 중괄호와 같은 줄에 선언
- 조건/반복문에 중괄호 필수

## 6. 줄바꿈(Line-wrapping)
- 최대 줄 너비: 120자
- package, import 선언문은 한 줄로
- 줄바꿈 후 추가 들여쓰기 필요

## 7. 빈 줄(Blank lines)
- package 선언 후 빈 줄 삽입
- import 선언 그룹 사이 빈 줄 삽입
- 메소드 사이 빈 줄 삽입

## 8. 공백(Whitespace)
- 줄 끝 공백 없음
- 중괄호 시작 전, 종료 후 공백
- 제어문 키워드와 소괄호 사이 공백
- 연산자 앞뒤 공백
- 콤마/세미콜론 뒤에만 공백
- 주석문 기호 전후 공백

## 도구 설정
- .editorconfig 파일로 기본 설정
- Checkstyle로 규칙 검사
- IDE별 설정 방법 제공 (Eclipse, IntelliJ, VI)
- Git 설정으로 일관된 코드 스타일 유지
